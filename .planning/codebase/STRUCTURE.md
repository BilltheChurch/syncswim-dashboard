## Directory Purposes

**`data/`:**
- Purpose: Root for all recorded sensor data; auto-created by scripts on first run
- Contains: One subdirectory per recording set
- Key files: Set directories named `set_NNN_YYYYMMDD_HHMMSS/`; never hand-edited

**`data/set_NNN_YYYYMMDD_HHMMSS/`:**
- Purpose: Atomic recording session; contains all CSVs and outputs for one button-press cycle
- Contains:
  - `imu_NODE_A1.csv` — IMU stream (columns: `timestamp_local, timestamp_device, node, state, set, ax, ay, az, gx, gy, gz`)
  - `vision.csv` — Vision stream (columns: `timestamp_local, frame, joint, angle_deg, visible, fps`)
  - `analysis.png` — Plot written by `analyze.py` after analysis
- Generated by: `recorder.py`, `vision.py`, or `sync_recorder.py` depending on phase

**`.planning/codebase/`:**
- Purpose: GSD codebase map documents consumed by planning and execution commands
- Generated: Yes (by gsd-codebase-mapper)
- Committed: Yes

**`.omc/`:**
- Purpose: Tool session and state management (not user-edited)
- Generated: Yes
- Committed: Unclear — treat as tool-managed

**`__pycache__/`:**
- Purpose: Python bytecode cache
- Generated: Yes
- Committed: No (should be in `.gitignore`)

## Key File Locations

**Entry Points:**
- `recorder.py`: BLE-only Phase 1 recorder — run with `python3 recorder.py`
- `vision.py`: Camera-only Phase 2 recorder — run with `python3 vision.py`
- `sync_recorder.py`: Primary Phase 3 dual-source recorder — run with `python3 sync_recorder.py`
- `analyze.py`: Post-hoc analysis — run with `python3 analyze.py [optional_set_dir]`
- `scan_ble.py`: BLE diagnostic — run with `python3 scan_ble.py`
- `receive_ble.py`: Legacy BLE diagnostic — run with `python3 receive_ble.py`

**Firmware:**
- `test_rec.ino`: Arduino sketch for M5StickC Plus2; flash via Arduino IDE

**Model Asset:**
- `pose_landmarker_lite.task`: MediaPipe pose landmarker model; required by `vision.py` and `sync_recorder.py`; loaded via `BaseOptions(model_asset_path=...)`; path resolved relative to `__file__`

**Documentation:**
- `task.md`: Phase-by-phase task checklist and hardware configuration reference
- `DEVLOG.md`: Technical decision log and problem/fix history

**Data Output:**
- `data/`: Auto-created on first run by any recording script; all CSV output lands here

## Naming Conventions

**Files:**
- Python scripts: `snake_case.py` — e.g., `sync_recorder.py`, `scan_ble.py`
- Arduino sketch: matches directory name — `test_rec.ino`
- Model files: descriptive with extension — `pose_landmarker_lite.task`
- CSV data files: `imu_NODENAME.csv`, `vision.csv` — fixed names within each set directory
- Set directories: `set_NNN_YYYYMMDD_HHMMSS` — zero-padded 3-digit set number + timestamp

**Python identifiers:**
- Classes: `PascalCase` — `State`, `SyncState`, `MjpegStreamReader`, `MyServerCallbacks`
- Functions: `snake_case` — `handle_notification`, `calc_angle`, `start_recording`
- Constants / config: `UPPER_SNAKE_CASE` — `TARGET_NAME`, `CHAR_UUID`, `BATCH_SIZE`, `READING_FMT`
- Module-level state instance: always named `state`

**Arduino identifiers:**
- Classes: `PascalCase` — `MyServerCallbacks`, `IMUReading`
- Global variables: `camelCase` — `deviceConnected`, `setNumber`, `batchIndex`
- Constants/defines: `UPPER_SNAKE_CASE` — `NODE_NAME`, `BATCH_SIZE`, `IMU_INTERVAL_MS`

## Where to Add New Code

**New recording source (e.g., second IMU node):**
- Model after the BLE handler pattern in `sync_recorder.py`
- Add fields to `SyncState` for the new source
- Add a new CSV writer pair (`open`/`close`) inside `start_recording()` / `stop_recording()`
- Add a new notification callback following `handle_ble_notification()`

**New analysis metric:**
- Add a function to `analyze.py` (e.g., `calc_gyro_magnitude()`)
- Add a new subplot to the `plt.subplots()` call in `analyze.py:main()`
- Output images save to `data/set_NNN_.../analysis.png` — keep that convention

**New diagnostic script:**
- Create a new top-level `.py` file (e.g., `check_latency.py`)
- Import `bleak` directly; use `asyncio.run(main())` pattern at bottom
- Do not add to `data/`

**Additional CSV columns:**
- IMU CSV schema is fixed by `recorder.py` / `sync_recorder.py` header row — update both the `writerow(["timestamp_local", ...])` call and the `load_imu()` dict keys in `analyze.py`
- Vision CSV schema is fixed similarly — update `vision.csv` header and `load_vision()`

**New firmware feature:**
- Edit `test_rec.ino`; BLE packet format changes require matching updates to `READING_FMT`, `READING_SIZE`, `HEADER_SIZE` constants in `recorder.py` and `sync_recorder.py`
