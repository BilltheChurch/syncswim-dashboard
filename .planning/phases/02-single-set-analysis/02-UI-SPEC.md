## Design System

| Property | Value |
|----------|-------|
| Tool | Streamlit native theming (`.streamlit/config.toml` `[theme]` section) — inherited from Phase 1 |
| Preset | not applicable |
| Component library | Streamlit built-in widgets + Plotly 6.6.0 for all charts and gauges |

**Note:** This is a Python/Streamlit project, not React. Design tokens are expressed as Streamlit theme config values and Plotly layout constants, not CSS custom properties. All tokens inherited from Phase 1 `01-UI-SPEC.md` unless explicitly overridden below.

## Spacing Scale

Inherited from Phase 1. No changes. Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Plotly annotation padding, gauge deduction label margins, inline badge margins |
| sm | 8px | Plotly subplot gaps, gauge internal padding, tab content top padding |
| md | 16px | Default Plotly chart margins (top, right), `st.columns` gap equivalent, tab panel padding |
| lg | 24px | Plotly chart margins (bottom with axis labels), section divider spacing between scoring card and tabs |
| xl | 32px | Plotly chart margins (left with Y-axis labels), fusion chart left margin for dual-axis labels |
| 2xl | 48px | Major section breaks (between scoring card header and tab content area) |

Exceptions:
- Gauge chart height: 200px (not a multiple of 4 — Plotly canvas requirement for readable gauge arcs)
- Phase timeline bar height: 40px (not a multiple of 4 — optimized for horizontal bar readability)
- Skeleton overlay frame display: 480px max-height (standard 16:9 frame constraint, not spacing-based)

### FINA Zone Color System (Phase 2 critical addition)

These three semantic colors are the core visual language for all scoring and quality indicators in Phase 2. They map directly to FINA artistic swimming execution deduction brackets.

| Zone | Color | Hex | Threshold | Deduction | Usage |
|------|-------|-----|-----------|-----------|-------|
| Clean | Green | `#09AB3B` | < 15 deg deviation | 0.0 | Gauge arc green zone, phase timeline green segments, keyframe angles within tolerance |
| Minor | Yellow | `#FACA2B` | 15-30 deg deviation | -0.2 | Gauge arc yellow zone, phase timeline yellow segments, keyframe angles with minor deviation |
| Major | Red | `#FF4B4B` | > 30 deg deviation | -0.5+ | Gauge arc red zone, phase timeline red segments, keyframe angles with major deviation |

**These colors are read from `config.toml` thresholds at runtime.** The hex values above are the Plotly rendering colors; the threshold boundaries come from `config["fina"]["clean_threshold_deg"]` and `config["fina"]["minor_deduction_deg"]`.

## Focal Point

| Tab | Primary Focal Point | Rationale |
|-----|-------------------|-----------|
| Persistent header (all tabs) | Overall execution score "X.X / 10" + 5 gauge row | The execution score is the single most important number. Coaches read the score first, then drill into detail tabs. Persistent display prevents context loss when switching tabs. |
| Tab 1: Overview (概览) | Phase timeline horizontal bar chart | After seeing the score, the coach asks "where did problems occur?" The color-coded timeline instantly answers this by highlighting which phase had issues. |
| Tab 2: Visual (视觉分析) | Keyframe comparison overlay (green vs red wireframe) | The visual comparison of ideal vs actual pose is the most intuitive coaching feedback. Coaches understand body positions better than numbers. |
| Tab 3: Sensor (传感器) | IMU + Vision fusion dual-axis chart | The fusion chart is the technical centerpiece of the project. It demonstrates sensor fusion visually — the core research contribution. |

### Scoring Card Persistence Decision

**Decision: Scoring card appears as persistent header ABOVE the tabs, visible on all tabs.**

Rationale: The 5 gauges arranged in a single row at approximately 200px height consume modest vertical space. They provide essential context regardless of which detail tab is active. A coach switching from Visual to Sensor tab should not lose sight of the overall scores. This follows the "dashboard header" pattern used by Catapult and Output Sports where KPI cards remain fixed while detail panels change below.

**Gauge Visual Specification:**

```python
go.Indicator(
    mode="gauge+number",
    value=metric_value,
    number={"font": {"size": 28, "family": "Source Sans Pro"}},
    title={"text": metric_name_chinese, "font": {"size": 14}},
    gauge={
        "axis": {"range": [0, max_value], "tickfont": {"size": 11}},
        "bar": {"color": "#0068C9"},  # Accent color for needle
        "steps": [
            {"range": [0, clean_threshold], "color": "#09AB3B"},    # Green zone
            {"range": [clean_threshold, minor_threshold], "color": "#FACA2B"},  # Yellow zone
            {"range": [minor_threshold, max_value], "color": "#FF4B4B"},  # Red zone
        ],
        "threshold": {
            "line": {"color": "#262730", "width": 2},
            "thickness": 0.75,
            "value": target_value,  # Standard template reference line
        },
    },
)
```

- Chart config: `{"displayModeBar": False}` to hide Plotly toolbar on gauges

## Plotly Chart Theme Usage

All charts in Phase 2 MUST apply `CHART_THEME` from `dashboard/components/__init__.py`. The standard application pattern:

```python
from dashboard.components import CHART_THEME

fig.update_layout(
    template=CHART_THEME["template"],
    font_family=CHART_THEME["font_family"],
    font_color=CHART_THEME["font_color"],
    paper_bgcolor=CHART_THEME["paper_bgcolor"],
    plot_bgcolor=CHART_THEME["plot_bgcolor"],
    margin=CHART_THEME["margin"],
)
```

Gauge charts override margins to `{"l": 0, "r": 0, "t": 24, "b": 0}` for compact display. Timeline chart overrides margins to `{"l": 0, "r": 0, "t": 0, "b": 0}` for zero-padding horizontal bar.
