# Phase 1 — UI Design Contract

> Visual and interaction contract for the Streamlit dashboard skeleton. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Streamlit native theming (`.streamlit/config.toml` `[theme]` section) |
| Preset | not applicable |
| Component library | Streamlit built-in widgets + Plotly 6.6.0 for charts |
| Icon library | Material Icons (Streamlit built-in `:material/` prefix) |
| Font | "Source Sans Pro" (Streamlit default — no override needed) |

**Note:** This is a Python/Streamlit project, not React. Design tokens are expressed as Streamlit theme config values and Plotly layout constants, not CSS custom properties.

---

## Streamlit Theme Config

Create `.streamlit/config.toml` with these values:

```toml
[theme]
primaryColor = "#0068C9"
backgroundColor = "#FFFFFF"
secondaryBackgroundColor = "#F0F2F6"
textColor = "#262730"
font = "sans serif"
```

| Token | Value | Role |
|-------|-------|------|
| `primaryColor` | `#0068C9` | Accent — interactive widgets, active nav items, toggle switches, primary buttons |
| `backgroundColor` | `#FFFFFF` | Dominant surface (60%) — main content area background |
| `secondaryBackgroundColor` | `#F0F2F6` | Secondary surface (30%) — sidebar background, card containers, expanders |
| `textColor` | `#262730` | All body text, labels, headings |
| `font` | `sans serif` | Maps to Source Sans Pro (Streamlit default) |

## Spacing Scale

Streamlit controls most layout spacing automatically. These values apply to **custom CSS injected via `st.markdown`** and **Plotly chart margins**.

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Plotly annotation padding, inline badge margins |
| sm | 8px | Plotly subplot gaps, compact metric card internal padding |
| md | 16px | Default Plotly chart margins (top, right), `st.columns` gap equivalent |
| lg | 24px | Plotly chart margins (bottom with axis labels), section divider spacing |
| xl | 32px | Plotly chart margins (left with Y-axis labels), page title bottom margin |
| 2xl | 48px | Major section breaks (between config categories on settings page) |

Exceptions: Streamlit's own widget spacing is not customizable without CSS hacks. Accept Streamlit defaults for widget-to-widget spacing. Only override via `st.markdown(unsafe_allow_html=True)` for the warning badge component.

## Typography

Streamlit controls base typography. These values define the **hierarchy used in `st.markdown`, `st.header`, and Plotly chart text**.

**Main type scale (4 sizes, 2 weights):**

| Role | Size | Weight | Line Height | Implementation |
|------|------|--------|-------------|----------------|
| Label | 14px | 400 (regular) | 1.4 | `st.caption` for secondary labels; Plotly axis titles |
| Body | 16px | 400 (regular) | 1.5 | Streamlit default `st.write` / `st.markdown` |
| Heading | 20px | 600 (semibold) | 1.2 | `st.subheader` for page sections |
| Page Title | 28px | 600 (semibold) | 1.2 | `st.header` or `st.title` — one per page only |

**Plotly chart text (scoped exception):** Plotly renders text into SVG/WebGL canvas where sub-14px sizes are standard practice for data-dense visualizations. The following sizes are scoped exclusively to Plotly `layout` and `trace` font properties and do NOT apply to any Streamlit widget or markdown text.

| Element | Size | Weight | Scope |
|---------|------|--------|-------|
| Chart title | 16px | 600 | Reuses main Body size — `layout.title.font.size` |
| Axis labels | 14px | 400 | Reuses main Label size — `layout.xaxis.title.font.size` |
| Axis tick labels | 11px | 400 | Plotly SVG only — `layout.xaxis.tickfont.size` |
| Annotation text | 12px | 400 | Plotly SVG only — `layout.annotations[].font.size` |
| Legend text | 12px | 400 | Plotly SVG only — `layout.legend.font.size` |

## Color

### Surface Distribution (60/30/10)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Main content area, page backgrounds, Plotly chart backgrounds |
| Secondary (30%) | `#F0F2F6` | Sidebar, `st.expander` backgrounds, settings form containers, metadata cards |
| Accent (10%) | `#0068C9` | Active nav highlight, coach/athlete toggle active state, primary action buttons, selected set indicator, config save button |
| Destructive | `#FF4B4B` | Streamlit's built-in error color — used only for `st.error` messages and the warning badge border on corrupted data |

### Accent Reserved For (explicit list)

- Active sidebar navigation item highlight
- Coach/Athlete role toggle active state
- "Save Settings" button on config page
- Selected set dropdown active border
- `st.spinner` loading indicator

### Semantic Colors for Data States

| State | Color | Usage |
|-------|-------|-------|
| Success / Complete data | `#09AB3B` | Green badge — set has both IMU and vision data |
| Warning / Partial data | `#FACA2B` | Yellow badge — set missing IMU or vision CSV |
| Error / Corrupted | `#FF4B4B` | Red badge — CSV parse error, unreadable data |
| Info | `#0068C9` | Blue — informational messages, index rebuild notification |

| Action | Confirmation Approach |

## Component Inventory (Phase 1)

### Sidebar Components

| Component | Streamlit Widget | Behavior |
|-----------|-----------------|----------|
| Role toggle | `st.sidebar.radio("视角切换", ["教练", "运动员"], horizontal=True)` | Stores in `st.session_state["role"]`; "团队同步" page hidden when role is "运动员" |
| Set selector | `st.sidebar.selectbox("选择训练组", options=session_names, placeholder="请选择一个训练组...")` | Populated from `sessions.json` index; stores path in `st.session_state["selected_set_dir"]` |
| Navigation | `st.navigation({...})` | 3 groups: 训练, 分析, 团队同步; Material Icons per page |

### Page: Training (训练监控)

| Component | Widget/Pattern | Content |
|-----------|---------------|---------|
| Page header | `st.header("训练监控")` | Displayed once at top |
| Set metadata card | `st.columns(4)` + `st.metric` per column | Shows: 训练组 #, 日期, 时长, 数据状态 — **primary focal point** |
| Data status badge | `st.success` / `st.warning` / `st.error` | Green/yellow/red based on data completeness |
| Empty state | `st.info` container | Shown when no set selected or no data directory |
| Placeholder sections | `st.container` with `st.caption` | "实时监控 — Phase 5 开发中" and "训练报告 — Phase 2 开发中" |

### Page: Analysis (数据分析)

| Component | Widget/Pattern | Content |
|-----------|---------------|---------|
| Page header | `st.header("数据分析")` | Displayed once at top |
| Placeholder sections | `st.container` with `st.caption` | "趋势追踪 — Phase 3 开发中" and "AI 分析 — Phase 4 开发中" |

### Page: Team (团队同步)

| Component | Widget/Pattern | Content |
|-----------|---------------|---------|
| Page header | `st.header("团队同步")` | Displayed once at top |
| Visibility gate | `st.Page(..., visibility="hidden")` when role is "运动员" | Coach-only page |
| Placeholder | `st.container` with `st.caption` | "团队同步分析 — Phase 6 开发中" |

### Settings (Sidebar Expander)

| Component | Widget/Pattern | Content |
|-----------|---------------|---------|
| Settings container | `st.sidebar.expander("设置", expanded=False)` | Collapsible settings form in sidebar |
| FINA threshold inputs | `st.number_input` per threshold | clean_threshold_deg, minor_deduction_deg, deduction values |
| Hardware config inputs | `st.text_input` per value | camera_url, ble_device_name, ble_char_uuid |
| Save button | `st.button("保存设置")` | Calls `save_config()`, shows `st.success("设置已保存")` |

## Interaction States

### Set Selector States

| State | Trigger | Visual |
|-------|---------|--------|
| No data directory | `data/` missing or empty | `st.info("暂无训练数据")` with empty state body copy |
| Sets available, none selected | App startup | Placeholder text in selectbox: "请选择一个训练组..." |
| Set selected, loading | User picks a set | `st.spinner("正在加载训练数据...")` |
| Set selected, complete data | Both IMU + vision CSVs present | Metadata card with green `st.success("IMU + 视觉数据完整")` |
| Set selected, partial data | Missing IMU or vision CSV | Metadata card with yellow `st.warning("部分数据缺失：...")` |
| Set selected, corrupted data | CSV parse error | `st.error("数据文件损坏：...")` with error state copy |

### Role Toggle States

| State | Visual Effect |
|-------|--------------|
| 教练 (Coach) | All 3 nav sections visible; "团队同步" page accessible |
| 运动员 (Athlete) | "团队同步" nav item hidden via `visibility="hidden"` |

### Config Settings States

| State | Visual |
|-------|--------|
| Expander collapsed | Single line "设置" with expand arrow |
| Expander open | Form fields populated from current `config.toml` values |
| Save clicked, success | `st.success("设置已保存")` toast — auto-dismisses |
| Save clicked, error | `st.error("设置保存失败：...")` with permission error details |
